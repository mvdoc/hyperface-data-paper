# Hyperface fMRI Dataset: Quality Assurance Report

This Jupyter Book contains quality assurance analyses for the Hyperface functional MRI dataset. The analyses include assessments of data quality through temporal signal-to-noise ratio (tSNR), motion parameters, inter-subject correlation (ISC), and outlier detection.

## Dataset Overview

The Hyperface dataset consists of functional MRI data collected while participants viewed dynamic facial stimuli. This comprehensive quality assurance report ensures the data meets high standards for reliability and scientific rigor.

## Data Organization

The dataset follows the Brain Imaging Data Structure (BIDS) specification:

- **`data/`**: Raw and preprocessed BIDS dataset (managed as git submodule/datalad dataset)
- **`data/derivatives/`**: BIDS derivatives including:
  - `fmriprep/`: Preprocessed fMRI data from fMRIprep
  - `freesurfer/`: FreeSurfer outputs
  - `qa/`: Quality assurance metrics and figures generated by this analysis pipeline

## Analysis Pipeline

The quality assurance pipeline includes:

1. **Motion Parameter Analysis**: Assessment of head motion during scanning
2. **Temporal SNR Computation**: Calculation of tSNR after confound regression
3. **Group tSNR Analysis**: Cross-subject tSNR comparison and visualization
4. **Inter-Subject Correlation**: ISC analysis to assess stimulus-driven responses
5. **Surface-based Analysis**: Visualization on fsaverage cortical surface
6. **Outlier Detection**: Identification of problematic scans or subjects

## Software Requirements

The analysis pipeline requires:

- Python 3.8+
- Key neuroimaging libraries: nibabel, nilearn, cortex
- Statistical libraries: numpy, scipy, pandas, scikit-learn
- Visualization: matplotlib, seaborn

Dependencies are managed using `uv` and specified in `pyproject.toml`.
